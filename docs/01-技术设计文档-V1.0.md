## 1. 项目定位与边界

### 1.1 产品定义

- **中文名称**: AI 教师工作空间
- **英文名称**: AI Teacher Workspace
- **版本**: V1.0
- **一句话定位**: 教师聊天式发送课件/教案/反思/计划总结，AI 自动识别类型、自动归档、自动关联；全校教师均可在线查看任意教师备课资料。

### 1.2 V1.0 明确不做

| 不做                | 原因             |
| ------------------- | ---------------- |
| 在线编辑 Word/PPT   | MVP 成本高       |
| 多人实时协同编辑    | 非 MVP           |
| AI 自动生成教案     | 质量与合规风险   |
| AI 自动生成反思     | 非 MVP           |
| AI 汇总集体备课意见 | V2 规划          |
| 复杂审批流          | 弱化表单         |
| 学生端 / 家长端     | 非本版本         |
| 移动端原生 App      | 仅预留小程序接口 |

### 1.3 核心业务对象

| 对象       | 说明                                         |
| ---------- | -------------------------------------------- |
| 学校       | 系统归属单位，含名称、简称、Logo             |
| 教研组     | 教师分组单位，首页按组织展示                 |
| 教师       | 系统用户，手机号为账号                       |
| 个人备课   | 教师本人提交的备课资料                       |
| 教学反思   | 必须关联一条个人备课的课后反思               |
| 集体备课   | 多人参与的备课记录，含评论/图片/附件         |
| 计划与总结 | 教学计划、教研计划、学期总结，按学年自动归档 |
| 附件       | Word/PDF/PPT/图片等文件资源                  |

## 2. 设计原则（DP）

| 编号   | 原则                      | 说明                                      |
| ------ | ------------------------- | ----------------------------------------- |
| DP-001 | 聊天是唯一新增入口        | 新增资料默认从聊天框进入，不填复杂表单    |
| DP-002 | AI 负责推荐，用户负责确认 | AI 识别类型/标题/关联，最终保存由教师确认 |
| DP-003 | 查看权限全校开放          | 所有教师可查看全校任意教师资料            |
| DP-004 | 编辑权限归属本人          | 教师只能新增/修改/删除自己的资料          |
| DP-005 | 默认在线预览              | Word/PPT/PDF 点击即看，下载为辅助         |
| DP-006 | 超过三次点击需重新设计    | 核心操作交互极简                          |
| DP-007 | 离职不删除资料            | 教师离职后首页隐藏，历史资料永久保留      |
| DP-008 | 业务页面统一              | 教师和管理员使用同一套业务页面            |
| DP-009 | 前后端分离                | Web/后台/小程序均调用同一套 API           |
| DP-010 | MVP 不追求大而全          | 只做上传/识别/归档/查看/预览/导出         |

## 3. 技术栈

### 3.1 总体选型

| 层级          | 方案                        | 版本 / 备注              |
| ------------- | --------------------------- | ------------------------ |
| **包管理**    | pnpm Workspace Monorepo     | pnpm >= 9                |
| **前端框架**  | Next.js (App Router)        | React 19 + TypeScript    |
| **UI 组件**   | Ant Design / Ant Design Pro | 后台表格场景成熟         |
| **后端框架**  | NestJS                      | TypeScript 模块化        |
| **ORM**       | TypeORM (Migration 模式)    | 禁用 `synchronize: true` |
| **数据库**    | PostgreSQL 15+              | 结构化业务数据           |
| **缓存/队列** | Redis 7+ + BullMQ           | AI/预览/导出异步任务     |
| **对象存储**  | 本地文件系统 / MinIO        | Adapter 抽象，可替换     |
| **在线预览**  | LibreOffice Headless        | Word→HTML, PPT→PDF       |
| **PDF 导出**  | Playwright                  | HTML 模板 → PDF          |
| **AI 模型**   | DeepSeek V4                 | Adapter 抽象，可替换     |
| **部署**      | Docker Compose              | 单校 MVP 部署            |

### 3.2 Adapter 抽象

所有外部依赖通过 Adapter 接口隔离，不直接绑定：

| 适配器包                     | 接口              | 默认实现     | 预留实现                |
| ---------------------------- | ----------------- | ------------ | ----------------------- |
| `@workspace/adapter-ai`      | `IAIAdapter`      | DeepSeek V4  | OpenAI / 本地模型       |
| `@workspace/adapter-preview` | `IPreviewAdapter` | LibreOffice  | OnlyOffice              |
| `@workspace/adapter-storage` | `IStorageAdapter` | 本地文件系统 | MinIO / S3 / 阿里云 OSS |

## 5. Adapter 抽象层设计

### 5.1 AI Adapter Interface

```typescript
// packages/adapter-ai/src/ai-adapter.interface.ts

export interface AIRecognitionInput {
  text?: string;
  fileName?: string;
  fileContent?: string; // 解析后的文本内容
  scope?: string; // 上下文范围
}

export interface AIRecognitionResult {
  predictedType: 'personal_lesson' | 'reflection' | 'group_lesson' | 'plan_summary' | 'unknown';
  title: string;
  confidence: number; // 0.00 ~ 1.00
  extractedFields: Record<string, any>;
  summary?: string;
}

export interface IAIAdapter {
  recognize(input: AIRecognitionInput): Promise<AIRecognitionResult>;
  recognizeWithContext(input: AIRecognitionInput, history: string[]): Promise<AIRecognitionResult>;
}
```

## 6. 服务架构与职责

### 6.1 服务矩阵

| 服务              | 端口 |               Queue 生产者               |    Queue 消费者    | 依赖适配器                       |
| ----------------- | ---- | :--------------------------------------: | :----------------: | -------------------------------- |
| `apps/api`        | 3000 | ai-recognition, file-preview, pdf-export |         —          | adapter-storage                  |
| `apps/web`        | 8080 |               — (通过 API)               |         —          | —                                |
| `worker-ai`       | —    |                    —                     | **ai-recognition** | adapter-ai                       |
| `worker-preview`  | —    |                    —                     |  **file-preview**  | adapter-preview, adapter-storage |
| `worker-export`   | —    |                    —                     |   **pdf-export**   | adapter-storage                  |
| `worker-schedule` | —    |                    —                     |    (Cron 定时)     | —                                |

### 6.2 关键原则

> **API 不允许同步执行耗时操作。所有 AI 识别、文件预览转换、PDF 导出必须通过 BullMQ 异步执行。**

流程：

```
用户请求 → API 校验权限 → 创建任务记录 (status: pending)
  → 入队 BullMQ → 立即返回 taskId
  → Worker 消费 → 处理完成 → 更新任务状态
  → 前端轮询/WebSocket 获取结果
```

### 6.3 BullMQ 队列设计

| 队列名           | Job 类型            | 生产者 | 消费者         | 并发 | 超时 |
| ---------------- | ------------------- | ------ | -------------- | :--: | ---- |
| `ai-recognition` | `classify-content`  | api    | worker-ai      |  3   | 30s  |
| `file-preview`   | `convert-word-html` | api    | worker-preview |  2   | 60s  |
| `file-preview`   | `convert-ppt-pdf`   | api    | worker-preview |  2   | 60s  |
| `pdf-export`     | `export-lesson-pdf` | api    | worker-export  |  1   | 120s |

### 6.4 worker-schedule 定时任务

| Job                     | Cron                         | 说明                           |
| ----------------------- | ---------------------------- | ------------------------------ |
| `term-switch-check`     | `0 0 * * *` (每天 0:00)      | 检查学年学期切换，触发系统通知 |
| `cleanup-preview-files` | `0 3 * * 0` (每周日 3:00)    | 清理过期的预览临时文件         |
| `archive-login-logs`    | `0 2 1 * *` (每月 1 号 2:00) | 归档登录日志                   |

### 7.2 ER 核心关系

```
School 1──N Department 1──N Teacher
Teacher 1──N Content
Content 1──N FileAsset (via lesson_attachment)
Content 1──N PreviewFile
Content 1──N AIRecognitionRecord
PersonalLesson 1──N Reflection
GroupLesson 1──N GroupLessonComment
Teacher 1──N AISession 1──N AIMessage
Teacher 1──N OperationLog
```

### 7.3 关键约束

- `teacher.mobile` UNIQUE
- `reflection.lesson_content_id` NOT NULL (教学反思必须关联个人备课)
- `content.deleted_at` 逻辑删除，默认 NULL
- `content.academic_year` + `semester` 系统自动计算，非人工输入
- 所有删除操作写入 `deleted_record` 表

### 7.4 Migration 管理

- 使用 TypeORM CLI 生成 Migration
- 禁用 `synchronize: true`，所有 Schema 变更通过 Migration 文件
- Migration 文件命名: `{timestamp}-{description}.ts`
- 开发环境: `pnpm run migration:run`
- 生产环境: Docker 容器启动时自动执行

### 8.2 错误码

| 错误码 | 含义                       | HTTP 状态码 |
| ------ | -------------------------- | :---------: |
| 0      | 成功                       |     200     |
| 40001  | 参数错误                   |     400     |
| 40100  | 未登录/Token 失效          |     401     |
| 40300  | 无权限                     |     403     |
| 40400  | 资源不存在                 |     404     |
| 40900  | 业务冲突 (如手机号重复)    |     409     |
| 50000  | 服务器异常                 |     500     |

## 9. AI 工作流

### 9.1 聊天识别全流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 教师上传  │───▶│ API 入队  │───▶│ Worker   │───▶│ 返回结果  │
│ 文件/文字 │    │ 创建记录  │    │ AI 识别  │    │ 确认弹窗  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                     │
                                          ┌──────────▼──────────┐
                                          │ 教师确认 / 修正      │
                                          │ [确认保存] [修改类型] │
                                          └──────────┬──────────┘
                                                     │
                                          ┌──────────▼──────────┐
                                          │ 写入 content + 业务表│
                                          │ 写入 ai_recognition  │
                                          │ 写入 operation_log   │
                                          └─────────────────────┘
```

### 9.2 AI 识别规则 (AIR)

| 编号    | 规则                                                                                          |
| ------- | --------------------------------------------------------------------------------------------- |
| AIR-001 | 仅识别四大类型: personal_lesson / reflection / group_lesson / plan_summary                    |
| AIR-002 | 输入支持: 文字、图片、Word、PDF、PPT                                                          |
| AIR-003 | 上下文范围: 单次会话，不跨无限历史                                                            |
| AIR-004 | 识别结果: 类型、标题、学年学期、关联对象，均允许修正                                          |
| AIR-005 | 识别失败: 弹出人工选择四大类型                                                                |
| AIR-006 | 反思关联: 识别为 reflection 时，查询最近 3 次个人备课供选择                                   |
| AIR-007 | 禁止自动关联: AI 不得自动绑定反思到具体备课                                                   |
| AIR-008 | 解析优先级: 文件名 > 文档标题 > 正文前段 > 上下文                                             |
| AIR-009 | 类型优先级: "反思"关键词 → reflection; "计划/总结" → plan_summary; PPT/课件 → personal_lesson |
| AIR-010 | 审计留痕: 每次识别保存输入摘要、结果、置信度、是否修正                                        |

### 9.3 教学反思关联流程

```
AI 识别为 "reflection"
  ↓
查询当前教师最近 3 次 personal_lesson (按 created_at DESC LIMIT 3)
  ↓
若无记录 → 提示"暂无个人备课，请先创建备课"
  ↓
若有记录 → 展示选择卡片 (标题 + 日期)
  ↓
教师选择一条 → 确认保存 → 写入 reflection 表 (lesson_content_id = 选中备课)
```

## 10. 文件与预览架构

### 10.1 上传流程

```
用户上传 → API 接收 multipart
  → adapter-storage.put() 保存原文件
  → 写入 file_asset 表 (status: uploaded)
  → 返回 fileId
  → 入队 file-preview (异步转换)
  → 写入 preview_file 表 (status: pending)
  → Worker 消费 → LibreOffice 转换
  → 更新 preview_file (status: success/failed)
```

### 10.2 预览方案

| 文件类型     | 转换引擎    | 输出            | 预览方式                |
| ------------ | ----------- | --------------- | ----------------------- |
| .doc / .docx | LibreOffice | HTML + 静态资源 | iframe 嵌入             |
| .ppt / .pptx | LibreOffice | PDF             | 浏览器 PDF viewer       |
| .pdf         | 无转换      | 原始文件        | 浏览器原生预览          |
| .jpg / .png  | 无转换      | 原始文件        | 图片查看器              |
| 其他         | —           | —               | 显示文件信息 + 下载按钮 |

### 10.3 失败处理

- 转换失败: 更新 `preview_file.status = 'failed'`，记录 `error_message`
- 前端显示: "预览生成失败，可下载原文件"，提供"重新转换"按钮
- 重试: POST `/api/preview/{fileId}/convert` 重新入队

## 11. 权限架构

### 11.1 权限矩阵

| 权限项               | 教师 | 管理员 | 实现                        |
| -------------------- | :--: | :----: | --------------------------- |
| 查看首页教师目录     |  ✅  |   ✅   | 无限制                      |
| 查看任意教师空间     |  ✅  |   ✅   | 无限制                      |
| 在线预览任意资料     |  ✅  |   ✅   | 无限制                      |
| 下载任意资料         |  ✅  |   ✅   | 无限制                      |
| 导出任意教师备课 PDF |  ✅  |   ✅   | 无限制                      |
| 新增本人资料         |  ✅  |   ✅   | owner check                 |
| 修改本人资料         |  ✅  |   ✅   | owner check                 |
| 删除本人资料         |  ✅  |   ✅   | owner check (逻辑删除)      |
| 新增他人资料         |  ❌  |   ✅   | PermissionGuard             |
| 修改他人资料         |  ❌  |   ✅   | PermissionGuard             |
| 删除他人资料         |  ❌  |   ✅   | PermissionGuard             |
| 查看后台管理入口     |  ❌  |   ✅   | 前端菜单 + 后端路由双重校验 |
| 组织/教师/学校管理   |  ❌  |   ✅   | role=admin                  |

### 11.2 实现策略

```typescript
// NestJS Guard 示例
@Injectable()
class OwnershipGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest();
    const user = request.user; // JWT 解析
    const targetTeacherId = request.params.teacherId || request.body.teacherId;

    // 管理员可操作所有
    if (user.role === 'admin') return true;

    // 教师只能操作自己的
    return user.teacherId === targetTeacherId;
  }
}
```
