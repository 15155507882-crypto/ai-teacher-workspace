# ADR-001：Workspace 作为系统核心对象

## 状态
已采纳

## 日期
2026-06-28

## 背景
V1 架构中，AI 工作台以 `ai_session`（会话）为中心。每次用户发消息创建一个 session，AI 返回结果后 session 即结束。文件上传、内容生成、预览修改都无法跨消息保持上下文。

## 决策
新增 `workspace` 作为系统核心对象。session 退化为 workspace 的子对象。

workspace 负责：
- 当前附件引用
- 当前生成内容快照
- 版本管理
- 上下文持久化
- 状态机

## 后果
- ✅ 文件上传后可跨消息继续聊天
- ✅ 页面刷新后可恢复工作状态
- ✅ 支持历史版本回退
- ⚠️ 需要新增 ai_workspace 表 + Redis 缓存双写

## 备选方案
- 继续以 session 为中心：无法解决上下文丢失问题，已否决
