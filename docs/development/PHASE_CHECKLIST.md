# 阶段检查清单 (Phase Checklist)

> **版本**：1.0.0 | **生效日期**：2026-06-28
> **依赖**：本文件服从 `DEVELOPMENT_CONSTITUTION.md`

---

## 使用说明

每个开发 Phase 必须按照以下清单逐项检查，全部通过后才能进入下一 Phase。

---

## 一、开发前检查（Pre-Development）

> 在写任何代码之前完成

- [ ] **C-01** 已阅读 `DEVELOPMENT_CONSTITUTION.md`
- [ ] **C-02** 已阅读相关 Spec 文件（ARCHITECTURE_SPEC / UI_SPEC / PROMPT_SPEC 等）
- [ ] **C-03** 已理解需求（PRD / Task 描述）
- [ ] **C-04** 已检查需求是否与宪法冲突
  - 如有冲突 → 停止 → 报告冲突分析 → 等待确认
- [ ] **C-05** 已输出开发方案：
  - 修改文件清单
  - 影响模块列表
  - 风险分析
  - 测试方案
- [ ] **C-06** 开发方案已获确认

---

## 二、开发中检查（During Development）

> 写代码过程中持续检查

- [ ] **C-07** 不边分析边改代码（先理解，再方案，再编码）
- [ ] **C-08** 不边写边改需求（需求变更 → 停止 → 更新文档 → 确认 → 继续）
- [ ] **C-09** 不做与本次任务无关的重构
- [ ] **C-10** 每次只改最小范围（与根因相关）
- [ ] **C-11** 涉及数据库变更时：
  - [ ] 已写 Migration 文件
  - [ ] 已说明迁移方案
  - [ ] 未使用 `synchronize: true`
- [ ] **C-12** 涉及架构变更时：
  - [ ] 已写/更新 ADR
  - [ ] 已更新 ARCHITECTURE_SPEC.md（如需要）
- [ ] **C-13** 涉及接口变更时：
  - [ ] 已更新 DTO 定义
  - [ ] 已更新接口文档（如前端需要适配）
- [ ] **C-14** 涉及状态机变更时：
  - [ ] 已更新 ADR-005
  - [ ] 已更新 CONVERSATION_SPEC.md
  - [ ] 已更新状态机测试用例
- [ ] **C-15** 涉及 Prompt 变更时：
  - [ ] 已在 prompt-registry.ts 中修改
  - [ ] 已更新版本号
  - [ ] 已运行 Prompt 测试

---

## 三、开发后检查（Post-Development）

> 代码写完后、声明完成前 — **最重要环节**

### 3.1 服务重启检查

- [ ] **C-16** 前端服务已重启（如有改动）
- [ ] **C-17** API 服务已重启（如有改动）
- [ ] **C-18** Worker 服务已重启（如有改动）
- [ ] **C-19** 确认新代码已生效（进程 PID + 启动时间 + 文件修改时间）

### 3.2 前端验收（如涉及页面）

- [ ] **C-20** 页面 URL 已确认
- [ ] **C-21** 当前渲染组件路径已确认
- [ ] **C-22** Network Request Payload 正确
- [ ] **C-23** Network Response Body 正确
- [ ] **C-24** Console 无报错
- [ ] **C-25** 页面截图已输出

### 3.3 后端验收（如涉及接口）

- [ ] **C-26** Controller 收到请求（日志确认）
- [ ] **C-27** DTO 参数正确
- [ ] **C-28** Service 执行日志正确
- [ ] **C-29** 返回结果 `code: 0` 且 data 正确

### 3.4 Worker/队列验收（如涉及异步任务）

- [ ] **C-30** API 入队 payload 正确
- [ ] **C-31** Worker job.data 正确
- [ ] **C-32** Worker 执行分支正确
- [ ] **C-33** 队列 waiting/active/failed 数量正常
- [ ] **C-34** Worker 执行日志正常

### 3.5 数据库验收（如涉及数据变更）

- [ ] **C-35** SQL 查询结果符合预期
- [ ] **C-36** 数据已正确写入/更新/删除
- [ ] **C-37** 未产生脏数据

---

## 四、Phase 完成输出（Phase Completion Report）

每个 Phase 完成后必须输出以下内容：

### 4.1 必填项

```markdown
## Phase {N} 完成报告

### 1. 修改文件
- 文件 1: {路径} — {修改说明}
- 文件 2: {路径} — {修改说明}

### 2. 新增接口
- {METHOD} {路径} — {说明}

### 3. 数据库变化
- 新增表: {表名} — {说明}
- 修改表: {表名} — {新增字段/修改字段}
- Migration: {文件名}

### 4. 状态机变化
- 新增状态: {状态名}
- 新增转换: {from} → {to}
- 无变化

### 5. 测试结果
- 单元测试: {通过数}/{总数}
- 接口测试: {通过数}/{总数}
- Worker 测试: {通过数}/{总数}

### 6. 兼容性
- 是否影响现有功能: 是/否
- 是否需要数据迁移: 是/否
- 是否需要前端同步部署: 是/否
```

## 五、AI 工作台专项检查（Task-Specific: AI Workspace）

AI 工作台相关开发需额外检查以下链路：

- [ ] **A-01** 前端 mode / text / file_id 参数正确
- [ ] **A-02** POST `/api/ai/chat` Request Payload 正确
- [ ] **A-03** Controller DTO 验证通过
- [ ] **A-04** Service 入队 payload 正确
- [ ] **A-05** Worker job.data 正确
- [ ] **A-06** `detectScene` 结果正确
- [ ] **A-07** 进入正确的场景分支（normal_chat 或业务场景）
- [ ] **A-08** 调用了 LLM（或正确跳过）
- [ ] **A-09** 生成了保存卡片（或正确跳过）
- [ ] **A-10** 写入了 content（或正确跳过）
- [ ] **A-11** 扣除了 quota（或正确跳过）
- [ ] **A-12** 页面最终显示结果正确

## 六、发布前检查（Pre-Release）

- [ ] **R-01** 所有测试通过
- [ ] **R-02** 无 Console Error / Warning
- [ ] **R-03** 文档已更新（ADR / Spec / PRD）
- [ ] **R-04** 数据库 Migration 已在 staging 环境执行
- [ ] **R-05** 环境变量已更新（如需要）
- [ ] **R-06** 回滚方案已确认

---

## 七、禁止跳过项

以下项目在任何情况下都不得跳过：

| 编号 | 项目                     | 原因                         |
| ---- | ------------------------ | ---------------------------- |
| C-01 | 阅读宪法                 | 最高准则不可跳过             |
| C-04 | 需求宪法冲突检查         | 防止违规开发                 |
| C-06 | 方案获确认               | 防止方向错误                 |
| C-16~19 | 服务重启确认         | 防止旧代码运行               |
| C-20~25 | 前端验收（如涉及）   | 防止"后端成功=前端成功"谬误 |
| C-26~29 | 后端验收（如涉及）   | 防止"code:0=业务正确"谬误   |
| C-35~37 | 数据库验收（如涉及） | 防止脏数据                   |
