# V2.2 风险点与回滚方案

## 1. 风险矩阵

| 风险                           | 影响 | 概率 | 缓解措施                               |
| ------------------------------ | :--: | :--: | -------------------------------------- |
| 数据库 Migration 失败          |  高  |  低  | 先备份、dry-run、提供回滚脚本          |
| shadcn/ui 引入破坏现有 CSS     |  中  |  中  | 逐步引入，先隔离测试                   |
| SSE 流式输出兼容性             |  中  |  中  | 保留普通接口 fallback                  |
| 文件预览需要后端转换           |  中  |  高  | 分阶段：先 PDF/图片，Word/PPT 文本预览 |
| API Key 加密/解密异常          |  高  |  低  | 使用标准 AES，预先生成测试用例         |
| 两栏布局影响现有数据流         |  低  |  低  | 先重构 UI，接口保持不变                |
| Drawer 替代 Modal 影响用户习惯 |  低  |  低  | 过渡期两种都支持                       |

## 2. 回滚方案

### 数据库回滚

```bash
# 执行回滚 Migration
pnpm run migration:revert

# 手动回滚（备用）
psql -U grandy -d ai_teacher << SQL
DROP TABLE IF EXISTS ai_call_logs;
DROP TABLE IF EXISTS school_ai_configs;
DROP TABLE IF EXISTS ai_providers;
ALTER TABLE personal_lesson DROP COLUMN IF EXISTS week_no;
ALTER TABLE group_lesson DROP COLUMN IF EXISTS week_no;
ALTER TABLE group_lesson DROP COLUMN IF EXISTS group_lesson_type;
ALTER TABLE reflection DROP COLUMN IF EXISTS match_score;
ALTER TABLE file_asset DROP COLUMN IF EXISTS parsed_text;
ALTER TABLE file_asset DROP COLUMN IF EXISTS preview_status;
SQL
```

### 页面回滚

- 每个重构页面保留原版本备份（如 `page.tsx.bak`）
- Git 分支保护：每个任务独立 commit，出问题 revert 单个 commit

### 依赖回滚

```bash
pnpm remove shadcn-ui @radix-ui/react-sheet lucide-react
pnpm install  # 恢复到 V2.1 状态
```

## 3. 安全风险

| 风险                  | 措施                                 |
| --------------------- | ------------------------------------ |
| API Key 明文泄露      | AES-256-CBC 加密存储，密钥在环境变量 |
| AI 调用日志泄露 Token | 不记录 API Key 明文，只记录调用信息  |
| 文件越权访问          | 下载和预览均校验 school_id + 角色    |

## 4. 测试备份

- 数据库备份：`pg_dump ai_teacher > backup_v2.2_pre.sql`
- 每次 Migration 执行前自动备份
