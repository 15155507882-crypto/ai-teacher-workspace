# V2.2 风险点与回滚方案

## 1. 风险矩阵

| 风险                   | 影响 | 概率 | 缓解                      |
| ---------------------- | :--: | :--: | ------------------------- |
| Migration 失败         |  高  |  低  | 备份 + dry-run + 回滚脚本 |
| shadcn/ui 破坏现有 CSS |  中  |  中  | 逐步引入                  |
| SSE 兼容性             |  中  |  中  | 保留普通接口 fallback     |
| 文件预览需后端         |  中  |  高  | 分阶段：先 PDF/图片       |
| API Key 加解密异常     |  高  |  低  | 标准 AES                  |

## 2. 数据库回滚

```bash
pnpm run migration:revert
```

## 3. 页面回滚

- Git 分支保护，每个任务独立 commit
- 可 revert 单个 commit

## 4. 安全

| 风险         | 措施                             |
| ------------ | -------------------------------- |
| API Key 泄露 | AES-256-CBC 加密，密钥在环境变量 |
| 日志泄露     | 不记录 Key 明文                  |
| 文件越权     | 校验 school_id + 角色            |
